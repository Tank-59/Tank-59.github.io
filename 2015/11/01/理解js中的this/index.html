理解js中的this

发表于 2015-11-01   |   分类于 JavaScript

this 是一个关键字，它代表函数运行时，自动生成的一个内部对象，只能在函数内部使用。

在函数定义的时候this的指向是不能确定的，只有函数执行的时候才能确定this到底指向谁，可以先理解为this指向的是那个调用它的对象。这样说虽然并不算严谨，但并不影响理解，从易到难，慢慢探究。

this仍然指向的是Obj。window是js中的全局对象，自己创建的变量实际上是给window添加属性。

如果一个函数中有this，但是它没有被上一级的对象所调用，那么this指向的就是window(非严格模式)；如果被上一级的对象所调用，那么this指向的就是上一级的对象；如果这个函数中包含多个对象，尽管这个函数是被最外层的对象所调用，this指向的也只是它上一级的对象。

构造函数中的 this

this对象指向p。因为 new 关键字创建了一个对象实例，改变this的指向。可以理解为将Fn复制里一份到对象p里，而此时只是创建，并未执行。而调用函数的是对象p，所以this即指向对象p。

函数中有 return 时，如果返回值是一个对象，那么this指向的就是那个返回的对象，如果返回值不是一个对象那么this还是指向函数的实例。（null对象除外）

除此之外，JS中还可以自行改变this的指向，用到的是call,apply方法。
